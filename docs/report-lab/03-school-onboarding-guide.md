# School Connect Report Lab School Onboarding Guide

## Purpose

This guide helps a school prepare the information and assets needed for Report Lab setup.

## What The School Must Provide Before Setup

Before onboarding begins, the school should provide:

- Official school name
- School code if already assigned
- School contact phone number
- School email address
- School address
- Logo or crest
- Motto
- Headteacher name
- Report signature names if used
- Current admission list or student register
- Class list
- Stream list
- Subject list
- Academic year and term dates
- Sample report layout preferences if the school has them

## Student List Requirements

Provide a clean student list with:

- Admission number
- Full name
- Gender if available
- Current class
- Current stream
- Active or inactive status
- Guardian contacts if available

Recommended format:

- One row per student
- One clear admission number per student
- No duplicate admission numbers

## Admission Number Requirements

Admission numbers must be:

- Unique within the school
- Consistent across the student register
- Easy for staff to type and verify

If the school already uses admission numbers, keep the same numbering style when possible.

## Class And Stream Structure

Provide the school's real structure before setup:

- Nursery classes if applicable
- Primary classes if applicable
- Secondary classes if applicable
- Streams for each class where required

Examples:

- Senior 1, stream A
- Senior 1, stream B
- Primary 7, stream A

## Subject List Requirements

Provide the subject list used by the school for report generation.

Include:

- Subject name
- Subject code if used
- Order or priority if the school has one
- Any optional subject/component rules

## Exam And Term Setup Requirements

Provide:

- Academic year name
- Term name
- Term start date
- Term end date
- Default assessment type
- Supported assessment types

## Marksheet Or Template Expectations

For digital import, prepare:

- CSV or Excel file with the correct column headers
- Consistent admission numbers
- Correct class, stream, subject, term, and exam type values

For scanned import, prepare:

- Clear scans or photos
- Good lighting
- Straight pages
- Visible marksheet header information

## School Branding Requirements

Provide the following branding assets and text:

- School name
- Logo or crest
- Motto
- Address
- Phone number
- Email address
- Website
- Stamp if supported
- Headteacher signature if supported

## Setup Timeline

Typical onboarding timeline:

| Phase | What happens | Estimated timing |
|---|---|---|
| Preparation | School sends student, class, stream, subject, and branding information | 1 to 3 days |
| Configuration | System setup and school structure provisioning | 1 to 2 days |
| Verification | Team checks import, report layout, and preview output | 1 to 2 days |
| Training | Staff receive user training | Same day or next day |
| First report cycle | School runs the first live cycle with support available | At the next suitable reporting window |

## Training Plan

Training should cover:

- Logging in
- Student management
- School structure
- Marks import
- Report preview
- Report printing
- Report verification
- Common mistakes

Recommended approach:

1. Admin training first
2. Data entry and marks import training next
3. Report release and verification training last

## First Report Cycle Checklist

- [ ] School account confirmed
- [ ] Students loaded

- [ ] Subjects confirmed
