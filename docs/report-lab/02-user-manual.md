# School Connect Report Lab User Manual

## Before You Start

You need a working school account and the correct school context. If you cannot see your school data, contact the SSAMENJ support team before continuing.

## 1. Logging In

1. Open the school application.
2. Go to the login page.
3. Enter your school email and password.
4. Enter the school code if the login form asks for it.
5. Sign in.

If login fails, check that:

- Your email is correct
- Your password is correct
- Your school account is active

## 2. Dashboard Overview

After login, the dashboard shows the main school workspace. From here you can move to:

- Students
- Reports
- Marks Import
- Marksheets
- Settings
- Release Center

Use the dashboard as the starting point for daily school work.

## 3. Managing Students

Use the Students area to:

- View enrolled students
- Add a new student
- Update student details
- Add guardian contacts
- Upload a passport photo
- Review student contact readiness

When adding a student:

1. Confirm the student full name.
2. Select the correct class.
3. Select the correct stream.
4. Enter or allow the system to generate the admission number.
5. Save the record.

Common mistakes:

- Choosing the wrong class
- Choosing the wrong stream
- Entering an admission number that already belongs to another student
- Forgetting to add a guardian contact

## 4. Setting Up Classes And Streams

School structure is managed in Settings under School Structure.

1. Open Settings.
2. Open School Structure.
3. Review the current school sections.
4. Confirm the canonical classes that belong to the school.
5. Add streams where needed.

Streams are usually coded by letters such as A, B, C, or D.

Important:

- Do not remove a section that already has students or marks without support guidance.
- Use the correct class and stream names that the school already uses.

## 5. Setting Up Subjects

The current system does not expose a separate subject-editing screen for daily staff. Subjects are normally prepared during school setup and provisioning.

What you should do:

1. Check that the expected subjects appear in report and import dropdowns.
2. Confirm that the subject list matches the school's real curriculum.
3. If a subject is missing, contact support or the onboarding team.

## 6. Marks Import

There are two main ways to import marks:

- Digital CSV or Excel import
- Scanned marksheet import with review

### 6.1 Uploading Marksheets Or Templates

From the Marks Import page, you can:

- Download the CSV template
- Download the Excel template
- Upload a completed file
- Paste CSV rows directly

### 6.2 Digital Import Steps

1. Open Marks Import.
2. Choose the Digital Marksheet option.
3. Download the template if needed.
4. Upload a CSV or Excel file, or paste CSV rows.
5. Click Dry run.
6. Review errors and warnings.
7. Fix the file if needed.
8. Click Commit valid rows only when the preview is correct.

### 6.3 Scanned Marksheet Steps

1. Open Marks Import.
2. Choose the Smart Marksheet Import option.
3. Upload a clear PNG, JPG, JPEG, WEBP, or PDF scan.
4. Confirm the class, stream, subject, term, and exam type.
5. Review the extracted rows.
6. Correct any rows that need review.
7. Commit only when you are satisfied with the results.

## 7. Reviewing Import Results

After a dry run, check:

- Total rows
- Valid rows
- Invalid rows
- Duplicate or protected rows
- Commit-ready rows

Look carefully for:

- Missing admission numbers
- Wrong class or stream
- Wrong subject
- Marks outside the valid range
- Duplicate lines

## 8. Fixing Import Errors

Use the error messages as your guide.

Typical fixes:

- Correct the admission number
- Correct the class or stream name
- Correct the subject name
- Make sure the mark value is valid
- Remove duplicate rows

If the error is about school setup, ask the onboarding team to check:

- The class list
- The stream list
- The subject list
- The active academic year and term

## 9. Generating Reports

1. Open Reports.
2. Choose the correct academic year, term, and class.
3. Choose the stream if needed.
4. Wait for the student list and report cards to load.
5. Open a student report to review the detail view.

The report is built from finalized marks only. If marks are missing, the report may show a missing-marks state.

## 10. Previewing Reports

The report page shows:

- Student list on the left
- Detailed preview on the right
- Report readiness
- Grade and ranking where enabled
- Branding and layout from school settings

Use the preview to confirm:

- Student names are correct
- Marks are correct
- The class and stream are correct
- The report looks ready for printing

## 11. Printing Reports

1. Open the student report preview.
2. Click Print / Save PDF.
3. Use your browser print dialog.
4. Save as PDF or print on paper.

If the layout looks wrong:

- Check the report personalization settings
- Check the browser zoom level
- Check the page orientation and margins in the print dialog

## 12. Verifying Reports

After a report is issued, it can be checked with a reference code on the public verification page.

You can also use the parent report link to confirm that the issued report opens correctly.

## 13. Settings

The Settings area is where administrators manage:

- School profile
- Academic year and term
- Report branding
- Marksheet settings
- OCR settings
- Grading scale
- Approval settings
- Appearance settings

Only users with the right access should change settings.

## 14. Human Approval Before Issuing Reports

Before issuing any report:

1. Review the report preview.
2. Confirm the marks are correct.
3. Confirm the branding is correct.
4. Confirm the student and class details are correct.
5. Only then issue the report link or release the report.

Do not skip this manual review.

## 15. Common User Mistakes

- Importing marks into the wrong term
- Choosing the wrong assessment type
- Forgetting to run the dry run before commit
- Uploading an unclear scan image
- Using the wrong class or stream
- Expecting a report to work before marks are finalized
