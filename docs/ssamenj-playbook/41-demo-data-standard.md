# Demo Data Standard

Demo and seed data must be useful, safe, and repeatable.

## Rules

- Use fake but realistic data.
- No real private student, customer, guest, tenant, owner, or staff data.
- Separate demo tenant/company/property from production data.
- Seeds must be repeatable.
- Add production wipe protection.
- Use Uganda-first examples where relevant.
- Use rentals examples where relevant for StayOS: properties, units, guests, bookings, owners, cleaners, maintenance, payments, and devices.

## Safety

Never seed secrets, real phone numbers, real payment details, or real identity data unless explicitly approved for a non-production demo environment.
